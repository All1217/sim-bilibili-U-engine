视频没有详细介绍conda环境，anaconda提供的虚拟环境和python3自带的venv不一样，你通过anaconda创建过的一系列虚拟环境通常存放在：C:\用户\你的用户名\anaconda3\envs。

一般不要随意切换虚拟环境，要么用python自带的，要么用其他产品（比如anaconda），而且还要注意有时候Pycharm会莫名奇妙帮你切换虚拟环境。

### 1.1 如何判断当前虚拟环境？

1. **查看 PyCharm 右下角**：会显示当前使用的解释器，例如：
   - `Python 3.9 (venv)`
   - `Python 3.11 (mycondaenv)`
   - `Python 3.8 (系统解释器)`
2. **查看 PyCharm 终端**：命令行提示符前面会有环境名称，例如：
   - `(venv) C:\project>`
   - `(mycondaenv) C:\project>`
3. **PyCharm 设置中查看**：
   - `File` → `Settings` → `Project` → `Python Interpreter`
   - 这里会明确显示当前使用的是哪个解释器

### 1.2 激活虚拟环境

在pycharm根据提示框引导选择虚拟环境，在创建项目时也可以改。或者打开命令行输入`python -m venv [项目名]`，会直接复制依赖并在本地创建目录。

## 2 如何复制环境？

主要是记录这个项目所处虚拟环境装过什么包。

### 2.1 法一

激活虚拟环境后，在根目录打开终端输入：`pip freeze > requirements.txt`。此时根目录出现名为`requirements.txt`的文件，该法会把所有相关或不相干的包记录进去

### 2.2 法二

安装`pip install pipreqs`，再在根目录运行`pipreqs . --force`，`--force` 会覆盖根目录已存在的 `requirements.txt`，也可以指定输出目录：`pipreqs src --force --savepath "$env:USERPROFILE\Desktop\requirements.txt"`（输出到桌面）
